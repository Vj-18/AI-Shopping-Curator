## 📌 What is Curatrix?

Curatrix simplifies online fashion shopping by combining product discovery, price comparison, and AI styling recommendations in one platform.

Users can search and filter products, compare prices across Myntra, Flipkart, and Amazon, get real-time outfit styling tips generated by the Claude API, manage a wishlist and cart with size selection, apply discount codes, and view their full order history — all behind a secure bcrypt-hashed login.

## ✨ Features

| Feature | Details |
|---|---|
| 🔐 Secure Auth | Signup and login with bcrypt-hashed passwords stored in SQLite |
| 🔍 Search & Filter | Search by name, filter by category (Shoes / Hoodies / Jackets / Dresses), sort by price |
| 🤖 AI Style Tips | Live Claude API call generates 3 outfit suggestions per product |
| 💰 Price Comparison | Compare prices across Myntra, Flipkart, Amazon — best deal highlighted automatically |
| ❤️ Wishlist | Save products with a heart icon, add to cart from wishlist |
| 🛒 Cart | Size selection, quantity control (+ / −), subtotal per item |
| 🏷 Discount Codes | Working codes: `CURATRIX10`, `FASHION20`, `NEWUSER15` |
| 📦 Order History | Every placed order saved to SQLite, viewable per user |
| 📷 Image Search | Upload a fashion image to search for similar styles |

## ⚙️ Setup & Installation

### 1. Clone the repository
```bash
git clone https://github.com/Vj-18/AI-Shopping-Curator.git
cd AI-Shopping-Curator
```

### 2. Install dependencies
```bash
pip install -r requirements.txt
```

### 3. Add your Anthropic API key

Create the secrets file:
```bash
mkdir .streamlit
```

Add this to `.streamlit/secrets.toml`:
```toml
ANTHROPIC_API_KEY = "sk-ant-your-key-here"
```

Get a free API key at [console.anthropic.com](https://console.anthropic.com)

> **Note:** The app works without an API key — it falls back to generic style tips automatically.

### 4. Run the app
```bash
streamlit run app.py
```

Open [http://localhost:8501](http://localhost:8501) in your browser.

## 🏗 Architecture

The app follows a single-file MVC-style pattern:

```
User Interaction
      ↓
Streamlit UI (View)
      ↓
Helper Functions + Session State (Controller)
      ↓
SQLite Database + Claude API (Model)
```

**Key design decisions:**

- **Session state** — Streamlit reruns the script on every interaction. `st.session_state` acts as a persistent key-value store for cart, wishlist, login status, and current page.
- **Dispatch table router** — Pages are mapped in a dictionary (`PAGES = {"home": page_home, ...}`) and called with `PAGES.get(page, page_home)()` — cleaner than if-elif chains and O(1) lookup.
- **bcrypt auth** — Passwords are hashed with a random salt on signup. Login uses `bcrypt.checkpw()` — plain text passwords are never stored or compared.
- **Parameterised SQL** — All queries use `?` placeholders to prevent SQL injection.
- **Graceful AI fallback** — Claude API call is wrapped in try/except; falls back to generic tips on any failure.

---

## 🔒 Security

- Passwords hashed with `bcrypt` (salted, slow hashing — resistant to brute-force and rainbow table attacks)
- API keys stored in `.streamlit/secrets.toml` — never hardcoded, never committed to Git
- SQL queries use parameterised placeholders — no SQL injection risk
- `.gitignore` excludes `curatrix.db` and `secrets.toml`

## 🚀 Deployment (Streamlit Cloud)

1. Push code to GitHub
2. Go to [share.streamlit.io](https://share.streamlit.io)
3. Click **New app** → select this repo → set main file to `app.py`
4. Under **Advanced settings → Secrets**, paste:
   ```toml
   ANTHROPIC_API_KEY = "sk-ant-your-key-here"
   ```
5. Click **Deploy**
